# Build Log: inkwell
**Date:** 2026-07-12
**Status:** deployed

## Idea Source
Researched. IDEAS.md was empty, so I surveyed the registry for uncovered problem
domains. The registry had no PDF *signing* tool (pdf-crush does compression only;
pagesmith — deployed concurrently by a sibling run — does page management). The sharpest
unshipped, highly-Googleable pain point is "sign PDF online free without uploading": people
routinely hand sensitive contracts to unknown SaaS uploaders. Client-side signing is the
honest answer. Rejected in-browser audio transcription — the local `scribewell`/`echoscribe`
dirs showed it had been attempted 4× and (at the time I chose) never shipped; a sibling run
later shipped scribewell during this build, confirming it was the right call to avoid.

## Tool Details
- **Name:** inkwell
- **Repo:** ben-gy/inkwell
- **Category:** document-tools
- **Audience:** Mainstream / small-business — someone needing to return a signed contract or
  form fast without uploading it, making an account, or installing anything.
- **Stack:** Vanilla TypeScript + Vite 6
- **Browser APIs:** pdf.js (pdfjs-dist) for rendering (own Web Worker), pdf-lib for writing,
  Pointer Events + Canvas 2D (pressure-sensitive signature drawing), Canvas rasterisation
  (typed signatures / checkmarks → transparent PNG), File API / Drag&Drop, Web Share, PWA.
- **Worker strategy:** pdf.js parses/rasterises in its bundled Web Worker (same-origin, no
  CDN). pdf-lib export runs on the main thread (fast for typical docs) with progress state.

## Privacy Model
- **Protected:** the PDF, the signature image, and all typed text — parsed, embedded and
  re-saved entirely in-tab. Nothing is persisted; reload wipes everything.
- **Not protected:** initial page load hits the GitHub Pages CDN (sees IP, never the doc);
  the native Share sheet forwards the file to a chosen app; the signature is *visible*, not
  a cryptographic/PKI signature.
- **Trust surface:** the static bundle over HTTPS + TLS to GitHub Pages. No third-party
  scripts, fonts, trackers, cookies or analytics.

## Architecture Decisions
- **Vanilla over React:** single workspace view, no deep component tree — a small observable
  store (`state.ts`) repainted incrementally is enough. Matches the rest of the factory.
- **Normalised annotation coordinates:** annotations are stored as 0..1 fractions of the
  *displayed* (rotation-aware) page, so they survive zoom and window re-render, and are only
  converted to PDF points at export.
- **Pure coordinate module (`coords.ts`):** the displayed→PDF mapping (top-left/y-down →
  bottom-left/y-up, plus page /Rotate) is the core risk, so it's a pure, DOM-free, fully
  unit-tested module. Derived and tested the pivot/rotate for all four rotations.
- **No signature persistence:** signatures live only in tab memory — a deliberate privacy
  decision (stronger threat-model story), also sidesteps the "localStorage is prefs-only" rule.

## Errors & Resolutions
1. **TS5.7 `Uint8Array<ArrayBufferLike>` vs `BlobPart`** — build failed on three
   `new Blob([bytes])` sites. Added a `bytesToBlob()` helper that copies into a fresh
   ArrayBuffer-backed array and routed all sites through it. Fixed.
2. **Preview tab hidden → pdf.js render hangs** — the preview renderer is a hidden tab, so
   `requestAnimationFrame` never fires and pdf.js's rAF-driven render loop stalled. Confirmed
   via `document.visibilityState === 'hidden'`. Test-harness workaround: shimmed rAF→setTimeout
   for verification. Not an app bug (real, visible browsers fire rAF normally).
3. **Class collision on `.ann-text` (real bug, fixed)** — the annotation container got class
   `ann ann-text` (from `ann-${kind}`), colliding with the inner editable's `.ann-text`.
   `querySelector('.ann-text')` matched the container, and although real typing worked (users
   click the inner div), it was a latent hazard. Renamed the inner editable to `.ann-field`
   across overlay.ts, workspace.ts and CSS; verified typed text now persists to the store and
   exports correctly.
4. **Checkmark default too small** — bumped default width from 0.035 → 0.05 of page width.
