## Tool Details
- **Name:** unHEIC
- **Repo:** ben-gy/unheic
- **Category:** image-tools
- **Audience:** Non-technical iPhone owners who need HEIC photos in a usable format and don't want to upload personal pictures to a stranger's server.
- **Stack:** Vite 6 + vanilla TypeScript + Vitest
- **Browser APIs:** libheif-js (WASM), Web Workers (module), OffscreenCanvas + convertToBlob, Transferable ArrayBuffer, Web Share, Clipboard (ClipboardItem), Service Worker (vite-plugin-pwa)
- **Worker strategy:** single dedicated module worker processing the queue sequentially (HEIC decode is memory-heavy; sequential avoids OOM on big batches)

## Privacy Model
- **Protected:** Photo pixels & every byte of the HEIC file; GPS/camera/timestamp EXIF (dropped on canvas re-encode); filenames. None transmitted.
- **Not protected:** Initial page + wasm load from GitHub Pages CDN (sees IP, not photos); whatever the user does with the output afterwards.
- **Trust surface:** Static bundle pinned per deploy; TLS to GitHub Pages; the libheif WASM shipped in-bundle (no third-party runtime CDN).

## Architecture Decisions
- Vanilla TS (no React) — a drop zone, gallery and modals don't need a component framework.
- `libheif-js/wasm-bundle` inlines the wasm (needs `'wasm-unsafe-eval'` in CSP) so there's no
  third-party runtime fetch — strengthens the privacy story.
- `optimizeDeps.exclude: ['libheif-js']` and `worker: { format: 'es' }` in vite.config for the
  wasm-in-worker path.
- fflate lazy-loaded only when "Download all (ZIP)" is used; ZIP built at store level (0) since
  images are already compressed.
