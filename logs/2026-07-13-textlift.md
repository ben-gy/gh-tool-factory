Verified Tesseract.js v7 (current, ~15-35% faster than v6) supports fully self-hosted operation.

## Tool Details
- **Name:** Textlift
- **Repo:** ben-gy/textlift
- **Category:** document-tools
- **Audience:** Non-technical people with a document problem — receipts, screenshots, scanned contracts — nervous about uploading them. Light, warm, spacious theme with a single teal accent.
- **Stack:** Vite 6 + vanilla TypeScript + Vitest
- **Browser APIs:** WebAssembly (SIMD, runtime-detected via wasm-feature-detect), Web Workers (Tesseract + pdf.js each in their own), Canvas 2D / createImageBitmap, Clipboard API (paste-to-ingest + copy out), IndexedDB (language cache), Web Share, Service Worker (PWA)
- **Worker strategy:** Tesseract.js spawns its own dedicated worker from self-hosted `worker.min.js`; pdf.js uses its bundled worker; main thread only orchestrates and paints progress

## Privacy Model
- **Protected:** files and extracted text never leave the device (no upload endpoint exists); English OCR works fully offline — verified by dry-running in a browser with no internet connectivity
- **Not protected:** non-English language models download once from jsDelivr (IP + language code visible, never file contents); disclosed in Threat Model
- **Trust surface:** GitHub Pages TLS + Tesseract.js/pdf.js/pdf-lib in the bundle + jsDelivr (optional) + Cloudflare Web Analytics beacon (disclosed)

## Architecture Decisions
- **Self-hosted Tesseract stack:** worker.min.js, SIMD + non-SIMD LSTM cores, and English traineddata (2.8 MB best_int) copied from npm packages at build time via vite-plugin-static-copy. Runtime SIMD detection picks the core file explicitly (the loader's directory-mode would prefer relaxed-SIMD variants we don't ship). PWA precaches SIMD core + English (~7.5 MB); the rare non-SIMD fallback is served but not precached.
- **Lazy rasterisation:** pages are rendered one at a time and each canvas freed after recognition — a 100-page scan never accumulates bitmaps. Only text, a 220 px thumbnail and per-page searchable-PDF bytes are kept. Page cap 100.
- **Searchable PDF via Tesseract's native `pdf: true` output** (per-page), merged with pdf-lib. Avoided hand-rolling an invisible-text overlay from word boxes.
- **pdf.js `intent: 'print'`** rendering (learned from pagewell/collate): schedules paint via microtasks, not requestAnimationFrame, so hidden/backgrounded tabs can't hang rendering.
- **pdf-lib gotcha:** `setProducer` is overwritten by pdf-lib on `save()`; branding lives in Creator instead (caught by test).
