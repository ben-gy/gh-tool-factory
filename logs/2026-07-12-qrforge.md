## Tool Details
- **Name:** QRForge
- **Repo:** ben-gy/qrforge
- **Category:** developer-utilities / privacy
- **Audience:** café/shop owners and privacy-aware people who won't route data through a QR-farm redirect
- **Stack:** vanilla TypeScript + Vite
- **Browser APIs:** BarcodeDetector, getUserMedia, Canvas 2D, Clipboard, Web Share, Service Worker
- **Worker strategy:** main-thread (generation and per-frame detection are fast; no CPU-heavy step)

## Privacy Model
- **Protected:** everything typed (WiFi passwords, contacts, links) and every scanned image / camera frame is processed on-device; codes are *direct* with no tracking-redirect middleman.
- **Not protected:** the QR code *is* the data (don't publicly post a code containing a secret); GitHub Pages logs the page load.
- **Trust surface:** the static bundle + TLS only. Zero runtime network calls — the CSP forbids `connect-src` beyond `self`. Works fully offline.

## Architecture Decisions
- Vanilla TS: two simple modes (generate/scan), no heavy state orchestration needed.
- `qrcode` (node-qrcode) for generation → crisp SVG + PNG; `jsqr` as a pure-JS scan fallback behind the native BarcodeDetector.
- Light, spacious, consumer-first theme per the factory's audience-first guidance (this is a mainstream consumer tool, not a dev/security one).
- Payload builders (WIFI/vCard/mailto/SMSTO/tel/geo) kept pure and unit-tested — escaping bugs there would silently break real WiFi joins / contact imports.
- Strict CSP with `connect-src 'self'` — no CDN, no model download, so the "nothing is uploaded" claim is airtight.
