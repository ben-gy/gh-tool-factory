## Privacy Model
- **Protected:** the mic stream is processed frame-by-frame on the audio thread and discarded —
  nothing is kept until record is pressed, and there is no upload endpoint in the code. Dropped files
  and exports never leave the device. `track.stop()` runs on stop, `pagehide` and `visibilitychange`,
  so the OS mic indicator goes out. Monitoring defaults to off. Works fully offline.
- **Not protected:** stated bluntly in the Privacy modal — **a shifted voice is not anonymity.** It
  defeats a casual listener; it does not defeat a determined analyst, because content, accent,
  cadence and background noise survive and pitch shifting is reversible. Anyone relying on this for
  safety should be told so, which is why it is the first bullet of the "Not protected" section rather
  than a footnote.
- **Trust surface:** the hash-pinned Pages bundle + TLS, the browser's Web Audio and decoders, the
  anonymous Cloudflare beacon, and feedback.benrichardson.dev only on pressing Send.

## Architecture Decisions

**A phase vocoder, written for this tool, rather than a library.** A granular/resample pitch shifter
is ~40 lines but moves pitch and formants together — that only ever gets you chipmunk. Sounding like
a different *person* needs the two separated, which needs frequency-domain work: 1024-point Hann STFT
at 4× overlap, per-bin true instantaneous frequency from the inter-frame phase advance, `k → k·p`
mapping with magnitude accumulation, integrated synthesis phases, overlap-add. Formants are then
handled by estimating the input's spectral envelope (a frequency-proportional moving average, O(N)
via a prefix sum) and multiplying each output bin by `E(j/s) / E(j/p)`. `s = p` gives gain 1 exactly
and the step is skipped; `s = 1` holds the formants still.

**Why a moving average and not a cepstrum.** A cepstral lifter needs two more FFTs per frame and a
log/exp pass. The prefix-sum envelope is O(N), needs no extra transform, and is easy to reason about
and test. The first attempt used a ±2-bin window, which was too narrow to erase the harmonics of a
low voice and partly undid the shift; widened to ±(5 + 0.08·bin).

**One graph builder for realtime and offline.** `buildChain()` takes a `BaseAudioContext`, so the
live mic path and the offline file render use literally the same nodes in the same order. It also
detects offline contexts (`'length' in ctx`) and sets parameters outright instead of ramping, so a
render is deterministic. This is what makes "preview == export" a structural guarantee rather than a
promise.

**A WaveShaper limiter, not a DynamicsCompressorNode.** The first version used a compressor, and
measurement in the browser showed "Natural" coming out 1.7 dB *louder* than it went in: Blink applies
an internal makeup gain, and by an engine-specific amount. Replaced with a soft-clip WaveShaper —
exactly transparent below 0.75, `tanh` above it — preceded by a ×0.5 pad so the curve's [−1, 1] input
domain covers signals up to ±2. Deterministic in every engine, and it took measured peaks from 1.19
(Alien, clipping) down to ≤1.0 everywhere while leaving Natural at exactly unity.

**No `requestAnimationFrame` anywhere.** The scope and timers run on `setInterval`. rAF never fires
in a hidden tab, and no functional path (recording, rendering, export) should be coupled to whether
the visuals happen to be animating — this has bitten previous builds in this factory.

**The worklet must be import-free.** AudioWorklet module resolution for relative specifiers is not
dependable across browsers, so the worklet goes through Vite's worker pipeline (`?worker&url`), which
bundles it into one self-contained ES module. Verified on the built artefact: `dist/assets/
voice-worklet-*.js` contains zero `import` statements and two `registerProcessor` calls.

## Test Results
- Tests written: 70
- Tests passed: 70
- Tests failed: 0

Substantive cases (not just plumbing):
- FFT checked against a naive O(n²) DFT to 8 decimal places; single-bin response for a pure tone;
  round-trip through the inverse; Hann mean-square = 3/8, which is what the overlap-add scaling
  derives from
- 440 Hz in → 880 Hz out at +12 semitones, 220 Hz at −12, and a musical fifth within 3% at +7
- Identity path within ±15% of unity gain
- Output bit-identical (to 6 dp) at block sizes 128 and 4096 — the engine cannot depend on the render
  quantum
- Latency is exactly 768 samples, matching the constant the offline renderer trims by
- `reset()` makes a second pass reproduce the first
- Formants: with formants held, f0 still doubles (autocorrelation, first-peak rule to avoid the
  octave-down error) while the spectral centroid is measurably lower than with them linked
- No non-finite sample under the most extreme warp (−18 st pitch, +12 st formant)
- WAV writer: header fields, both length fields, sample round-trip, clamping, NaN/Infinity → silence,
  empty buffer
- Limiter curve: transparent below the knee, monotonic, bounded, continuous across the knee

Four initial failures, all fixed: three were wrong test expectations (RMS of a unit sine is 1/√2 not
0.5; a denormal ≈ −2e−21 is not `Object.is` zero; `toPcm16` deliberately maps ±Infinity to silence),
and one was a genuinely wrong-headed test — asserting formant preservation on a *pure tone*, which is
degenerate (preserving the envelope of a sine means putting the energy back where it started, so it
fights the shift). Replaced with a harmonic-voice probe measuring f0 by autocorrelation.

## Errors & Resolutions
1. **A stale service worker served the previous tool's shell.** The preview tab at :5199 opened with
   the title "Meshprep" — a known trap, since every tool in this factory shares that port. Fixed for
   the session by unregistering all service workers and clearing all caches before each reload.
2. **TypeScript 5.7 typed-array generics.** `Float32Array` from a function signature is
   `Float32Array<ArrayBufferLike>`, which no longer satisfies DOM methods wanting
   `Float32Array<ArrayBuffer>` (`copyToChannel`, `getFloatTimeDomainData`, `WaveShaperNode.curve`).
   Fixed with explicit `Float32Array<ArrayBuffer>` annotations at the four boundaries.
3. **DynamicsCompressorNode added ~1.7 dB of engine-specific makeup gain** — see the limiter decision
   above. Caught only because the browser dry-run measured the output peak against the input peak.
4. **Envelope smoothing too narrow at low frequencies**, weakening formant preservation for low
   voices. Widened from ±2 bins to ±(5 + 0.08·bin).
5. **Measurement lag in the dry-run itself**, not the tool: preset changes are debounced 320 ms
   before rendering, so an immediate `!rendering` check returned the *previous* clip and made every
   preset look one step behind. Worth recording because the readings looked like a real bug for
   several minutes.
6. Playing a take back while starting a new recording would have shared the elapsed-time readout
   between two different clocks; recording now stops playback first.
