# Build Log: Collate (PDF page organiser)
**Date:** 2026-07-12
**Status:** built_and_deployed_then_unpublished_duplicate

## Summary of the run

This run built two tools to completion before shipping a final product, because the first two
ideas each turned out to duplicate an existing tool — the second duplication only became visible
*after* deployment because a concurrent factory run shipped the same tool mid-build.

1. **echoscribe** (in-browser Whisper transcription) — abandoned at the build/verify stage on
   discovering `scribewell` already covers it (in the registry + a live repo). Removed the local
   dir; logged its genuine improvements to `EXPANSION_IDEAS.md` under `scribewell` (mic capture,
   JSON export, editable transcript, windowed streaming progress).
2. **collate** (merge/reorder/rotate/delete PDF pages) — built, tested, deployed. Then found that
   a **concurrent run had shipped `pagesmith`** (identical tool) live at
   `pagesmith.benrichardson.dev` — it did **not** exist when I checked repos at the start of the
   collate build (verified absent via `gh repo list` and `gh repo view`). Collate is therefore a
   duplicate and was **un-published** (Cloudflare DNS record deleted). The private repo
   `ben-gy/collate` and the local build are kept pending a human keep-or-delete decision.

## Why not simply delete collate

Deleting a GitHub repo is hard to reverse and this is a non-interactive run, so per the standing
guidance ("for hard-to-reverse or outward-facing actions, confirm first unless durably
authorised") I took only the reversible step: removed the `collate.benrichardson.dev` DNS record
to stop the outward-facing duplication. Reviving collate = re-add the CNAME; fully retiring it =
delete `ben-gy/collate`. That choice is left to the user.

## Collate tool details (the built artifact)
- **Repo:** ben-gy/collate (private, un-indexed)
- **Category:** document-tools / pdf
- **Audience:** someone assembling signed contracts / scans / statements into one PDF who won't
  upload sensitive documents to a web service
- **Stack:** Vanilla TypeScript + Vite
- **Browser APIs:** pdf.js (pdfjs-dist, worker parse + canvas thumbnails), pdf-lib (assemble),
  Canvas 2D, File System Access API, Drag & Drop, Web Share, Web Workers
- **Worker strategy:** pdf.js parses in its own worker; thumbnails rasterised on the main thread
  at low scale as a background task decoupled from the interaction/export lock

## Privacy model
- **Protected:** every PDF added + the output PDF — parsed, rendered and assembled entirely in the
  tab; zero runtime network calls (pdf.js + pdf-lib are bundled).
- **Not protected:** only the initial page load (GitHub Pages logs it).
- **Trust surface:** the static bundle + TLS to GitHub Pages. No third-party CDN at runtime.

- **Real workflow dry-run:** generated a valid 2-page PDF in-page, dropped it → parsed (2 pages)
  → clicked Export → captured the produced blob → confirmed a valid `%PDF-1.7` document ending in
  `%%EOF`, 842 bytes. End-to-end merge/assemble path verified in the browser.
- Discovered (and fixed) that pdf.js pauses `page.render` in a hidden tab (waits on
  requestAnimationFrame). Fixes: thumbnails render off the `busy` lock (so export is never
  blocked), each render has a cancel-on-timeout that reports a `ThumbTimeoutError`, and a
  `visibilitychange` listener resumes rendering when the tab is shown. Thumbnails show as
  skeletons only in the always-hidden preview tab; they render normally on a visible tab.

## Recommendation for the user
`pagesmith` already fills the PDF-page-organiser niche and is live + indexed. Either:
- **Keep pagesmith, retire collate:** `gh repo delete ben-gy/collate` (repo is private, unlisted).
- **Prefer collate:** re-add the Cloudflare CNAME (`collate` → `ben-gy.github.io`, DNS-only),
  retire pagesmith, and consider folding collate's hidden-tab-safe thumbnail rendering into it.

## Root cause / process note
Two concurrent factory runs raced on overlapping ideas. The duplicate check at Step 2 passed
because pagesmith's repo did not yet exist when collate started. A cross-run lock (or a shared
"in-flight ideas" list checked immediately before `gh repo create`) would prevent this class of
collision.
