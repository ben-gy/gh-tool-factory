**Date:** 2026-07-16

## Idea Source
Researched / gap analysis. IDEAS.md was empty. Diffed `tools/` against `registry.json` and the
ben-gy org first (per the deploy-backlog memory): the only unregistered local builds were `collate`
and `pagewell` — both duplicate PDF-page tools (same job as the registered `pagesmith`), the classic
concurrent-run collision. They already exist as live repos, so they were left alone rather than
promoted as duplicates or destructively deleted.

Then surveyed the catalog for a genuine gap. The 18 registered tools cover documents, images, media
(record/transcribe/GIF), encryption/privacy, files, P2P and dev utilities — but there was **no audio
editing tool at all**. "Trim MP3 online" / "make a ringtone" is a high-volume, privacy-sensitive
search where every existing result demands an upload and slaps on a watermark. Clipwell fills that
gap with the Web Audio API as the central pillar.

## Tool Details
- **Name:** Clipwell
- **Repo:** ben-gy/clipwell
- **Category:** media-tools (index: media)
- **Audience:** podcasters/musicians clipping a private recording; anyone making a ringtone from a
  song they own without uploading it.
- **Stack:** Vanilla TypeScript + Vite 6
- **Browser APIs:** Web Audio (`decodeAudioData`, `OfflineAudioContext`, `AudioBufferSourceNode`,
  `GainNode`), Canvas 2D, Web Workers, Web Share, Service Worker (PWA)
- **Worker strategy:** single dedicated encoder worker (LAME MP3 via `@breezystack/lamejs` +
  hand-rolled WAV), channel PCM transferred in, progress via postMessage.

## Privacy Model
- **Protected:** source file, decoded PCM, trimmed region and exported clip never leave the device;
  no upload endpoint exists. No account, no cookies for data, no watermark. Works offline.
- **Not protected:** the exported clip is an ordinary unencrypted audio file; decode support depends
  on the browser's native codecs (honest error otherwise).
- **Trust surface:** static bundle + TLS to GitHub Pages; the browser's Web Audio decoder; the
  bundled LAME encoder (runs locally in a worker); a cookie-less Cloudflare Web Analytics beacon for
  anonymous page views (audio never sent to it).

## Architecture Decisions
- **Vanilla TS, not React** — a single imperative workspace around a canvas; no multi-pane state to
  justify React.
- **`@breezystack/lamejs`** for MP3 — a pure-JS LAME port, so no WASM and, crucially, no COOP/COEP
  cross-origin-isolation headers (which GitHub Pages can't set). WAV is a hand-rolled 16-bit PCM
  encoder, guaranteed-available fallback and the target of the heaviest unit tests.
- **OfflineAudioContext for rendering** — sample-accurate fades (linear gain ramps) and normalise
  gain applied natively while bouncing only the selected region; keeps the export path independent of
  `requestAnimationFrame` (which is throttled in a backgrounded/hidden tab — see the rAF-hang memory).
- Pure logic (DSP peaks/normalise/region math, WAV encoder, formatting) split into standalone modules
  so it's fully unit-testable without a DOM or Web Audio.
